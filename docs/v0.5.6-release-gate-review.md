### T006 Mount Plan Contract

状态：通过。

命令：

```bash
skillrun consumer mount plan --client <id> --json
skillrun consumer mount plan --client <id> --config <path> --json
```

证据：

- 输出 `consumer.mount_plan.v1`。
- 支持 `claude-desktop` 和 `cursor` plan adapter。
- unsupported client 返回 `unsupported-client` warning。
- unparseable config 返回 warning 且 `changes=[]`。
- 已有 config 只输出 SkillRun server entry before/after 摘要，不回显完整 client config。
- 测试证明 plan 不创建、不修改真实 config 文件。
- 输出 `router-runtime-not-implemented` warning，避免误读为 Router 已可运行。
- 未实现 apply / rollback。

## 边界复审

v0.5.6 没有引入：

- Desktop / Tauri。
- `skillrun ui`。
- long-running daemon。
- Router runtime。
- MCP client config mutation。
- `.skr import`。
- signed package trust。
- dependency installation / vendoring。
- OS sandbox。
- marketplace。

需要特别保留的措辞：

- Registry 是 inventory，不是 trust store。
- Switchboard `enabled=true` 是 future exposure intent，不是 trusted/runnable。
- `.skr` 是 import/distribution artifact，不是 MCP runtime entry。
- Run history 是 evidence-first；Desktop 应消费 Core JSON surfaces，不应直接读取 `.skillrun/runs`。
- Mount plan 是 preview，不是 apply。

## Release Decision

当前建议：

- 可以把 `codex/v0.5.6-release-polish` 作为 v0.5.6 release candidate。
- 合并 main 前再跑一次最终 release validation。
- 合并 main 后按 `docs/release-checklist.md` 顺序处理 main CI、tag、GitHub Release 和 release notes 回写。

不建议在 v0.5.6 继续新增：

- `consumer runs inspect`
- `consumer mount apply`
- `consumer mount rollback`
- Router runtime
- daemon
- Desktop UI
