# v0.5.6 Mount Plan Contract Review

## 一句话判断

T006 可以进入设计冻结，但不应该在 v0.5.6 做 apply / rollback；一键挂载的第一性约束是 plan-first：先让用户和 Desktop 看清楚将修改什么，再谈自动写配置。

## 关键洞察

### 1. 挂载的是 Router，不是 `.skr` 或单个 capsule

SkillRun 的 MCP client 挂载入口应该是未来 Router / Gateway：

```bash
skillrun router serve --mcp
```

而不是：

- 把 `.skr` 直接写进 Claude Desktop / Cursor。
- 把每个 capsule path 分别写成一个 MCP server。
- 让 Desktop 直接拼接 `serve --mcp --cwd <capsule>`。

原因很简单：MCP client 只应该知道一个稳定本地入口；具体暴露哪些 tools 应由 `switchboard`、`consumer exposure --json` 和 readiness gate 决定。

### 2. Mount plan 是配置变更 diff，不是安装器

`consumer mount plan` 的职责只有：

- 检测目标 client 是否可能存在。
- 找到候选配置路径。
- 读取当前配置。
- 计算写入 SkillRun Router 后的结构化变更。
- 生成 backup path 建议。
- 输出 warnings。

它不应该：

- 创建、修改或删除真实 MCP client 配置。
- 启动 Router。
- 导入 `.skr`。
- 安装依赖。
- 认为配置存在就等于 client 正在运行。

### 3. v0.5.6 不应承诺具体客户端路径长期稳定

Claude Desktop、Cursor 或其他 MCP client 的配置位置会随产品版本变化。SkillRun 可以维护检测规则，但 JSON contract 不应该把某个路径写成协议事实。

因此输出应区分：

- `client`: 用户请求的客户端 id。
- `detected`: 当前机器是否检测到候选配置或应用痕迹。
- `config_path`: 当前 plan 选中的候选路径。
- `path_source`: `detected` / `default` / `override`。
- `warnings`: 检测不确定、文件不存在、JSON 无法解析、client unsupported 等。

### 4. plan 结果必须可解释、可拒绝、可测试

好的 mount plan 输出应该能支撑三个消费者：

- CLI 用户：看懂会改哪一个文件。
- Desktop UI：展示 diff 和风险提示。
- 自动化测试：固定 JSON shape，不依赖用户机器实际安装 Claude/Cursor。

所以 v0.5.6 的实现如果要做，也应该支持 test override，例如后续可以通过 `--config <path>` 或环境变量指定配置文件，避免测试依赖真实用户配置。

## 风险 / 反例 / 误区

### R001 把 mount plan 做成“偷偷 apply”

如果 `plan` 为了用户体验顺手写配置，就会把安全叙事打穿。

`plan` 必须是只读；真实写入应是未来 `apply`，并且需要：

- preview diff。
- backup。
- rollback。
- 失败恢复。
- 明确提示正在修改第三方 client 配置。

### R002 把每个 capsule 写进 MCP client

这会让一键挂载绕过 SkillRun 的核心控制面：

- `switchboard` 失效。
- `consumer exposure` 失效。
- readiness gate 被客户端配置固化。
- 后续 Router / Desktop 很难统一管理。

正确模型是一个 SkillRun MCP server，内部动态暴露 enabled + ready 的 Manifest-derived tools。

### R003 过早支持太多客户端

第一版不应该追求 Claude Desktop、Cursor、Windsurf、VS Code、Cherry Studio、Continue 全覆盖。

T006 只需要定义 client registry 模型，最多选择 1-2 个高价值客户端作为首批 adapter。其他 client 返回 `unsupported-client` 也比写错配置更好。

### R004 输出 raw config 造成泄露

MCP client 配置可能包含其它 server 的 token、env 或 command args。

Mount plan 不应默认回显完整原配置。它应输出结构化 `changes`，只展示 SkillRun 相关 entry 的 before/after 摘要。

## 更强版本

### 推荐命令

```bash
skillrun consumer mount plan --client claude-desktop --json
skillrun consumer mount plan --client cursor --json
```

如果要让测试和高级用户可控，后续可以增加：

```bash
skillrun consumer mount plan --client claude-desktop --config <path> --json
```

但 `--config` 是 plan 输入 override，不是 stable client discovery contract。

### 推荐 JSON 合同

```json
{
  "command": "consumer mount plan",
  "schema_version": "consumer.mount_plan.v1",
  "client": {
    "id": "claude-desktop",
    "name": "Claude Desktop",
    "supported": true,
    "detected": true
  },
  "operation": "install_or_update_router",
  "config": {
    "path": "<display-path>",
    "path_source": "detected",
    "exists": true,
    "parseable": true
  },
  "backup": {
    "path": "<display-path>.skillrun.bak",
    "required_before_apply": true
  },
  "router": {
    "server_name": "skillrun",
    "command": "skillrun",
    "args": ["router", "serve", "--mcp"]
  },
  "changes": [
    {
      "kind": "upsert_mcp_server",
      "server_name": "skillrun",
      "before": null,
      "after": {
        "command": "skillrun",
        "args": ["router", "serve", "--mcp"]
      }
    }
  ],
  "warnings": []
}
```

### 错误 / 降级合同

Unsupported client：

```json
{
  "command": "consumer mount plan",
  "schema_version": "consumer.mount_plan.v1",
  "client": {
    "id": "unknown",
    "name": "unknown",
    "supported": false,
    "detected": false
  },
  "operation": "install_or_update_router",
  "config": null,
  "backup": null,
  "router": {
    "server_name": "skillrun",
    "command": "skillrun",
    "args": ["router", "serve", "--mcp"]
  },
  "changes": [],
  "warnings": [
    {
      "code": "unsupported-client",
      "message": "client is not supported by mount planning"
    }
  ]
}
```

Unparseable config：

```json
{
  "config": {
    "path": "<display-path>",
    "path_source": "detected",
    "exists": true,
    "parseable": false
  },
  "changes": [],
  "warnings": [
    {
      "code": "unparseable-config",
      "message": "config exists but is not valid JSON; plan cannot compute a safe patch"
    }
  ]
}
```

## 是否进入 v0.5.6 实现

判断：T006 可以在 v0.5.6 实现 plan-only，但只在满足以下约束时实现：

- 只支持 `plan`，不支持 `apply` / `rollback`。
- 只输出 SkillRun server entry 的 before/after，不输出完整 client config。
- client discovery 可降级，不因未安装客户端失败。
- unsupported client 返回结构化 warning，而不是 panic 或硬失败。
- 测试不依赖真实用户机器配置。

如果实现成本开始扩张到真实客户端兼容矩阵，就停止在 contract review，不进入实现。

## 实现状态

v0.5.6 已实现：

```bash
skillrun consumer mount plan --client <id> --json
skillrun consumer mount plan --client <id> --config <path> --json
```

实现边界：

- 只支持 `plan`，不支持 `apply` / `rollback`。
- 不创建、修改或删除真实 MCP client 配置。
- 支持 `claude-desktop` 和 `cursor` 的 plan adapter；其他 client 返回 `unsupported-client` warning。
- `--config <path>` 作为 plan 输入 override，用于测试和高级用户，不是 stable client discovery contract。
- 只输出 SkillRun server entry 的 before/after 摘要，不回显完整 client config。
- 若配置不可解析或 `mcpServers` shape 不可安全 patch，输出 warning 且 `changes=[]`。
- plan 目标仍是未来 `skillrun router serve --mcp`；v0.5.6 输出 `router-runtime-not-implemented` warning，避免误读为 Router 已可运行。

## 当时建议

1. 将本文作为 T006 contract review 结果。
2. 保持 v0.5.6 的 mount 范围为 plan-only `consumer mount plan --client <id> --json`。
3. `apply` / `rollback` 留到 v0.6 Desktop alpha，且必须先有 backup 与 rollback contract。
