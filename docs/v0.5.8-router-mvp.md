# v0.5.8 Router MVP

## 一句话判断

v0.5.8 应该只实现最小 SkillRun Router：一个本地 MCP stdio 入口，按 registry + switchboard + readiness 快照暴露 enabled 且 ready 的 Skill Capsule；不做 daemon、不写 MCP client 配置、不做热加载。

## 背景

v0.5.6 已经实现：

- `consumer inventory --json`
- `consumer exposure --json`
- `consumer runs list --json`
- `consumer mount plan --client <id> --json`

v0.5.7 已经把 README、官网、Desktop handoff 和 release 叙事收束为 public surface。

现在阻塞 Desktop 一键挂载的不是 UI，而是缺少真实 runtime entry：

```bash
skillrun router serve --mcp
```

没有 Router，`consumer mount plan` 只能预览未来配置，Desktop 也只能展示“将来可以挂载”。有了 Router，MCP client 配置里只需要一个稳定本地入口，具体暴露哪些 tools 由 registry、switchboard 和 readiness 决定。

## 目标

v0.5.8 的目标是让 mount plan 指向的命令真实可运行：

```bash
skillrun router serve --mcp
skillrun router serve --mcp --dry-run
```

Router MVP 应满足：

- 从本机 registry 读取 capsules。
- 只暴露 `enabled=true` 且 readiness `ok=true` 的 capsule。
- 对每个暴露 capsule 在启动快照中运行 Consumer Mode Manifest validation。
- MCP `tools/list` 返回所有可暴露 tool。
- MCP `tools/call` 按 tool name 路由到对应 capsule。
- MCP resources 仍只暴露 SOP 和 examples，不暴露 action source、`.skillrun/` 内部目录或 registry 文件。
- `--dry-run` 输出结构化 JSON，供 Desktop / CI 检查。

## 非目标

v0.5.8 不做：

- Desktop / Tauri。
- `skillrun ui`。
- long-running daemon API。
- HTTP / SSE / Streamable HTTP MCP transport。
- MCP client config apply / rollback。
- `.skr import`。
- marketplace。
- signed package trust。
- dependency installation / vendoring。
- OS sandbox。
- hot reload。
- profile / workspace / multi-user permission model。

## 启动快照语义

Router MVP 是 snapshot router，不是 daemon registry watcher。

启动时：

1. 读取 registry。
2. 过滤 enabled + ready capsules。
3. 对每个 capsule 执行 Consumer Mode validation。
4. 检查 tool name 唯一性。
5. 构建 MCP server state。

启动后：

- registry 或 switchboard 改变不会自动热加载。
- 用户需要重启 Router 才能刷新 exposed tools。
- 单次 tool call 的 dependency failure 必须返回结构化 MCP tool error，并保持 Router 存活。

## 失败边界

Router 必须 fail closed：

- enabled capsule Manifest stale / invalid：不得暴露。
- tool name 重复：启动失败，避免 ambiguity。
- registry 中 disabled capsule：不得暴露。
- readiness false：不得暴露。
- tool call 找不到 tool：返回 JSON-RPC invalid params。

## 成功标准

- `skillrun router serve --mcp --dry-run` 输出 `router.mcp.v1` contract。
- dry-run 中 tools 来自 enabled + ready capsules。
- disabled capsule 不出现在 Router tools 中。
- stdio `tools/list` 能列出 registry/switchboard 暴露的 tool。
- stdio `tools/call` 能路由并产生正常 run record。
- mount plan 不再输出 `router-runtime-not-implemented` warning。
- README / release notes 明确 v0.5.8 添加的是 Router runtime MVP，不是 daemon 或 config mutation。

## 实现状态

v0.5.8 已完成并合入 `main`：

- `skillrun router serve --mcp` 提供本地 MCP stdio Router runtime。
- `skillrun router serve --mcp --dry-run` 输出 `router.mcp.v1` 结构化合同。
- Router 按 registry + switchboard + readiness + Consumer Mode Manifest validation 构建启动快照。
- mount plan 已指向真实 Router entry，不再输出 `router-runtime-not-implemented` warning。
- README、release notes 和 docs index 已明确 v0.5.8 是 Router runtime MVP，不是 daemon、Desktop 或配置写入。

## 后续演进

- Router 仍是 snapshot router；hot reload、process management 和 daemon API 留给后续版本。
- Desktop 应消费 Router dry-run / consumer JSON surfaces，而不是读取 `.skillrun/` 或 registry 内部文件。
