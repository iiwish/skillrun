# v0.5.12 Capsule Import

## 一句话判断

v0.5.12 应补齐 `.skr import` 的最小 Core 合同：把 `.skr` 作为分发物导入到本机 SkillRun 管理目录，并登记到 registry，但默认不启用、不安装依赖、不声称信任。

## 背景

v0.5.8 到 v0.5.11 已经给 Desktop 准备了 Router、Safe Mount Apply、run list 和 run inspect。进入 Desktop 之前还缺一个关键入口：

```bash
skillrun import <package.skr> [--id <id>] [--to <dir>] [--json]
```

Desktop 不应该直接操作散落的 capsule 文件夹，也不应该把 `.skr` 直接挂给 MCP client。`.skr` 是 import/distribution artifact；Router 才是 MCP runtime entry。

## 核心原则

### 1. Import is not install

`skillrun import` 只负责解包、静态校验和 registry 登记。它不安装 Python/Node 依赖，不创建虚拟环境，不下载包，不修改 MCP client 配置。

### 2. Import is not trust

导入 `.skr` 不表示该 capsule 可信。导入后的 capsule 默认 `enabled=false`，必须由后续 `switchboard enable <id>` 显式启用，并继续经过 readiness gate。

### 3. Core owns archive safety

Desktop 不应该自己解 tar。Core 必须拒绝绝对路径、`..`、Windows prefix、空路径和会逃逸目标目录的 archive entry。

### 4. Registry remains the control plane

导入成功后，capsule 进入本地 registry，`source_type` 为 `imported_skr`。后续 `consumer inventory`、`consumer exposure`、Router 和 switchboard 只通过 registry 观察它。

## 目标

- 增加 `skillrun import <package.skr> [--id <id>] [--to <dir>] [--json]`。
- 默认导入到 SkillRun 管理目录下的 `capsules/<id>`。
- 解包到临时目录，校验 `.skillrun/manifest.generated.yaml` 和 source hash 后再移动到最终目录。
- 默认 registry id 来自 Manifest `skill.name`；`--id` 可覆盖。
- 导入成功后登记 registry，`enabled=false`，`source_type=imported_skr`。
- JSON 输出 `import.v1`，供 Desktop 调用。

## 非目标

- 不做 signed package trust。
- 不做 package registry / marketplace。
- 不做 dependency installation。
- 不做 `switchboard enable`。
- 不做 MCP client mount。
- 不做 package update / reinstall。
- 不做 import from URL。
- 不做 package manifest 格式升级。

## JSON 合同

### Success

```json
{
  "command": "import",
  "schema_version": "import.v1",
  "ok": true,
  "package_path": "<path>",
  "registry_path": "<path>",
  "capsule": {
    "id": "refund",
    "path": "<skillrun-home>/capsules/refund",
    "source_type": "imported_skr",
    "enabled": false
  },
  "warnings": [
    {
      "code": "not-enabled",
      "message": "Imported capsule is disabled until switchboard enable is called."
    },
    {
      "code": "dependencies-not-installed",
      "message": ".skr import does not install runtime dependencies."
    }
  ]
}
```

### Failure

失败时 v0.5.12 使用 stderr + non-zero exit code。后续 Desktop 如果需要统一失败 JSON，再增加 structured error，而不是把未稳定错误字段提前冻结。

## 成功标准

- 导入 pack 生成的 `.skr` 后，`registry list --json` 能看到 `source_type=imported_skr` 且 `enabled=false`。
- 导入路径内没有 `.skillrun/runs` 或 `dist`。
- 导入不执行 action source。
- 路径穿越 archive 被拒绝，并且不会在目标目录外写文件。
- 重复 registry id 被拒绝，不覆盖已有导入目录。
