# v0.5.13 Import Router Contract

## 一句话判断

v0.5.13 应硬化 Desktop 最关键的消费链路：`.skr import` 只进入本地 inventory，默认不暴露；只有显式 `switchboard enable` 且 readiness ok 后，Router 才能把 imported capsule 暴露成 MCP tool。

## 背景

v0.5.12 已经提供：

```bash
skillrun import <package.skr> --json
```

但 Desktop 真正依赖的不是单个 import 命令，而是一条端到端合同：

```text
.skr package
  -> skillrun import
  -> registry source_type=imported_skr enabled=false
  -> consumer exposure empty
  -> router dry-run empty
  -> switchboard enable
  -> consumer exposure has tool
  -> router dry-run / stdio can list and call tool
```

这条链路必须由 Core 测试锁住，否则 Desktop 很容易重新读取内部目录或自行推断暴露状态。

## 核心原则

### 1. Import remains non-exposure

导入只改变本地 inventory，不改变 MCP exposure。`enabled=false` 是 import 的安全默认值。

### 2. Switchboard remains the exposure gate

`switchboard enable <id>` 仍是显式启用动作；启用前 `consumer exposure` 和 Router 都不得暴露 imported capsule。

### 3. Router consumes Core state only

Router 只消费 registry + switchboard + readiness + Consumer Mode validation。Desktop 不应该读取 `.skillrun/manifest.generated.yaml`、`.skillrun/runs` 或 registry 文件来判断暴露状态。

### 4. Imported run evidence stays local to imported capsule

通过 Router 调用 imported capsule 时，run record 应写入 imported capsule 的 `.skillrun/runs`，而不是作者原始 capsule 或 package 目录。

## 目标

- 增加 import → inventory → exposure → switchboard → router dry-run → router stdio call 的端到端测试。
- 明确 imported capsule 默认 hidden。
- 明确 enable 后 Router 暴露来自 imported capsule path。
- 明确 Router call 会在 imported capsule 下产生 run evidence。

## 非目标

- 不新增 CLI 命令。
- 不新增 daemon API。
- 不做 Router hot reload。
- 不做 package update/reinstall。
- 不做 Desktop UI。
- 不做 signed package trust。

## 成功标准

- `skillrun import <package.skr> --json` 后，`consumer inventory --json` 显示 `source_type=imported_skr` 且 `enabled=false`。
- import 后、enable 前，`consumer exposure --json` 为空。
- import 后、enable 前，`router serve --mcp --dry-run` 为空。
- `switchboard enable <id>` 后，`consumer exposure --json` 和 Router dry-run 都暴露同一个 tool。
- Router stdio `tools/call` 能调用 imported capsule，并在 imported capsule 目录写入 `mode=mcp` 的 run record。
