# v0.5.11 Runs Inspect

## 一句话判断

v0.5.11 应补齐 `consumer runs inspect <run-id> --json`：让 Desktop Envelope Explorer 能通过 Core 查看单次 run evidence，而不是直接读取 `.skillrun/runs` 内部目录。

## 背景

v0.5.6 已经实现：

```bash
skillrun consumer runs list --json [--capsule <id>] [--limit <n>]
```

`runs list` 解决的是发现问题：有哪些 registered capsule 的 run evidence。Desktop 还缺一个稳定 detail 查询面：

```bash
skillrun consumer runs inspect <run-id> --json [--capsule <id>]
```

## 核心原则

### 1. Core reads evidence, Desktop consumes contract

Desktop 不应读取 `.skillrun/runs/<run-id>/record.json`、`output.json`、`stdout.log` 或 artifact 文件。Core 读取本地 evidence 后输出稳定 JSON。

### 2. Inspect is not export

`inspect` 可以返回 record summary、output/error envelope、artifact metadata 和 log availability，但默认不返回完整 input、stdout/stderr content 或 artifact content。

### 3. Registry scope remains the boundary

v0.5.11 只检查当前 registry 中已登记 capsule 的 run evidence。未登记 capsule 的历史 run 不属于这个 consumer control-plane surface。

## 目标

v0.5.11 的目标：

- 增加 `skillrun consumer runs inspect <run-id> --json [--capsule <id>]`。
- 输出 `consumer.runs.inspect.v1`。
- 当 run id 在多个 registered capsules 中出现时 fail closed，要求传入 `--capsule <id>`。
- 对 missing / invalid record 或 envelope 返回结构化 JSON，而不是让 Desktop 读取文件兜底。
- 默认不返回完整 input、stdout/stderr content 或 artifact content。

## 非目标

v0.5.11 不做：

- `--include-input`。
- artifact content read。
- stdout/stderr content read。
- global run index。
- retention / cleanup。
- run search / filter language。
- Desktop / Tauri。
- daemon API。

## 成功标准

- `consumer runs inspect <run-id> --json --capsule <id>` 返回单次 run evidence。
- `inspect` 输出 envelope，但不输出 input、stdout/stderr content 或 artifact content。
- missing / invalid evidence 返回 JSON warning/status，不要求 Desktop 读内部文件。
- ambiguous run id 返回结构化 `AmbiguousRunId`。
- `cargo test` 和 `cargo clippy --all-targets -- -D warnings` 通过。
