# SkillRun v0.4.3: CI And Runtime Error Stabilization

**版本**：v0.4.3 patch release

**状态**：Ready_For_Release

**更新日期**：2026-05-14

## 背景

v0.4.2 合入 `main` 后，GitHub Actions 中 `Rust CI / clippy` 和 `Rust CI / test` 失败。`fmt` 通过，说明问题集中在 Linux runner 上的 Rust 检查和测试行为，而不是格式化。

v0.4.3 是一个窄范围 patch release，目标是恢复 `main` 的 CI 可信度，并让缺失 Node/Python metadata runtime 时的错误文案跨平台稳定。

## 修复内容

- 修复 Linux clippy 下 fake Python helper 的 unused variable warning，避免 `cargo clippy --all-targets -- -D warnings` 在 Ubuntu runner 上失败。
- 明确 CI `test` job 的 Python、Node 和 Pydantic 依赖，避免测试隐式依赖 GitHub runner 镜像预装内容。
- 将 Node/Python metadata extractor 启动失败时的 `NotFound` 错误规范为 `program not found`，让 Windows 和 Linux 的测试断言不依赖底层 OS 文案。
- 将 GitHub Actions `actions/checkout` 更新到 v6，消除 Node 20 action deprecation 路径上的旧依赖。

## 边界

- 不新增 runtime feature。
- 不改变 Manifest schema。
- 不新增 adapter。
- 不改变 v0.4.2 official reference capsules 的产品范围。
- 不发布 registry、installer、signed capsule 或 runtime image。

## 验证

- `cargo fmt --check`
- `cargo clippy --all-targets -- -D warnings`
- `cargo clippy --target x86_64-unknown-linux-gnu --all-targets -- -D warnings`
- `cargo test`
- GitHub Actions `Rust CI` on `main` after v0.4.2 CI fix merge: passed.
