# v0.5.5 Core Contract Hardening

## 一句话判断

v0.5.5 是 SkillRun 进入 Desktop / v0.6 前的内核合同硬化版本，不做新场景扩张，优先把 Manifest、schema、adapter lifecycle 这些长期边界打磨到稳定。

## 背景

v0.5.4 已经完成 Consumer Mode、registry、switchboard、MCP stdio、pack、run record 和 release 验证基线。当前核心风险不在功能缺口，而在同一份运行事实被多个模块重复解释：

- Manifest 仍由多个模块直接读取 `serde_yaml::Value`。
- Core schema validator 与 adapter validator 的支持范围不完全一致。
- adapter timeout 只结束父进程，尚未形成清晰的进程生命周期合同。
- 公开 Manifest 中仍有早期占位文案，影响 IR 质感。

## 目标

v0.5.5 的目标是让 SkillRun 的内核更像一个 Manifest-driven runtime，而不是一组可用但分散的 CLI 能力。

## 非目标

- 不引入 Desktop / Tauri。
- 不做 marketplace。
- 不扩大 sandbox 或权限系统叙事。
- 不把 SkillRun 改造成通用 Agent framework。
- 不新增复杂 workflow 编排能力。

## Work Items

### C001 Schema Contract Unification

Core validator 必须覆盖官方模板与 adapter validator 已经声明支持的 JSON Schema 子集。

验收标准：

- Core 支持 `minLength`。
- Core 支持 `minimum`。
- Core 不再静默接受未知 `type`。
- 相关行为有单元测试或集成测试覆盖。

### C002 Manifest Public IR Quality

Manifest 作为运行 IR，不应暴露内部 milestone 或早期占位内容。

验收标准：

- `sop_summary` 来自 `SKILL.md` 的稳定文本摘要，或使用中性的 fallback。
- Manifest 中不出现类似 `T004` 的内部任务文案。
- 现有 Manifest 合同测试继续通过。

### C003 Typed Manifest Access Boundary

减少 runtime、consumer、readiness、MCP、pack、registry 对 Manifest YAML 的重复手动解释。

验收标准：

- 新增或规划共享 Manifest access layer。
- 至少一个高频路径迁移到共享 access layer。
- 保持 Consumer Mode fail-closed 行为不变。

### C004 Adapter Process Lifecycle

明确 adapter timeout 对子进程树的处理边界。

验收标准：

- 至少为 command adapter 增加 timeout residual-process 风险测试或平台化处理。
- 文档明确 SkillRun 的 lifecycle 控制边界，不把它包装成 sandbox。

## 当前优先级

第一阶段先做 C001 与 C002。它们风险低、收益高，能直接改善“可验证”和 Manifest IR 质量。

## 当前进展

- C001 已完成：Core schema validator 支持 `minLength`、`minimum`，并拒绝未知 `type`。
- C002 已完成：Manifest `sop_summary` 不再暴露内部 milestone 占位文案。
- C003 已推进：新增 `manifest_access` 共享读取层，并迁移 runtime、consumer、readiness、MCP、pack、registry、inspect、permissions 的 YAML Manifest 读取路径。runtime、MCP 与 readiness 已开始使用 `ManifestView` 领域字段读取接口。
- C004 已推进：adapter timeout 逻辑收敛到共享 process helper，Windows 使用 `taskkill /T /F`，Unix 使用独立进程组的 `TERM`/`KILL` 作为 best-effort 进程树终止。
- C005 已完成：runtime 与 readiness 对缺失 `runtime.adapter` / `runtime.entrypoint` 的 Manifest 保持一致，执行前 fail-closed，不创建 run record。
- C006 已完成：schema contract validation 被提升为 Manifest 生成、readiness 与 runtime 的共享前置合同，非法 schema 不再延迟到 Action 执行阶段才暴露。
- C007 已完成：新增 `docs/v0.5.5-release-gate-review.md`，用命令合同矩阵确认 Consumer Mode 执行、MCP 暴露和 `.skr` 分发共享 Manifest 静态合同。
