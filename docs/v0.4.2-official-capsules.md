## 一句话判断

v0.4.2 交付一组官方参考胶囊，用低风险、高频场景展示 SkillRun 的核心价值：把脆弱 prompt 里的工具使用规则，移进可检查、可测试、可运行拦截的 Skill Contract。

## 范围

v0.4.2 新增三个官方参考 capsule：

- `examples/commit_message_gate`
- `examples/bounded_file_patcher`
- `examples/readonly_diagnostics_runner`

它们不是 marketplace、registry、adapter collection 或安全产品。它们是可运行示例，用来证明 SOP、schema、preflight、permissions、artifact 和 run record 如何一起约束 Agent-callable action。

## Capsule 1: Commit Message Gate

### 业务问题

Agent 经常生成过长、格式不稳定或不符合团队规范的 commit message。如果团队只把规则写进 prompt，模型很容易漂移。

### SkillRun 价值

`commit_message_gate` 把 Conventional Commits 规则放进 preflight。默认只验证 message 并返回 normalized subject；只有显式 `perform_commit=true` 时才尝试对已 staged changes 执行 `git commit -m`。它不会运行 `git add .`。

### 证明点

- schema 限定输入形态。
- preflight 拒绝非法格式、过长 subject 和 multiline message。
- artifact 记录验证结果。
- 默认路径无 Git 副作用。

## Capsule 2: Bounded File Patcher

### 业务问题

Agent 修改文件时容易整文件覆盖、改错目录或碰到敏感配置。仅靠 prompt 约束很脆弱。

### SkillRun 价值

`bounded_file_patcher` 只允许在 `src/`、`docs/`、`tests/` 下做一次精确 old-text/new-text replacement。它拒绝绝对路径、`..`、隐藏路径、lockfiles、package manifests 和 secret-like 文件。

### 证明点

- 路径边界进入 preflight。
- 修改必须有 exact old_text match。
- ambiguous replacement 被拒绝。
- 生成 markdown patch artifact。
- 文档明确它不是 OS sandbox。

## Capsule 3: Read-only Diagnostics Runner

### 业务问题

开发者希望 Agent 能跑一些诊断命令，但把任意 shell 交给模型风险过高。

### SkillRun 价值

`readonly_diagnostics_runner` 不接受任意命令字符串，只接受 schema 中的 named diagnostics：`pwd`、`list`、`git_status`。运行时不用 `shell=True`，记录 stdout、stderr、exit code 和 timeout。

### 证明点

- 使用 allowlist，而不是黑名单。
- schema 本身移除了 shell 注入空间。
- 输出被截断，避免污染上下文。
- artifact 记录诊断结果。

## 使用路径

每个官方胶囊都可以按同一条路径检查：

```bash
skillrun manifest --cwd examples/commit_message_gate
skillrun inspect --cwd examples/commit_message_gate
skillrun check --cwd examples/commit_message_gate
skillrun test --cwd examples/commit_message_gate
skillrun serve --mcp --cwd examples/commit_message_gate --dry-run
skillrun pack --cwd examples/commit_message_gate
```

其他两个 capsule 将 `--cwd` 换成对应目录即可。

## 非目标

- 不提供 registry 或 store。
- 不提供 `skillrun install`。
- 不承诺安全下载或签名验证。
- 不支持 arbitrary shell。
- 不把文件 patcher 声称为 sandbox。
- 不在 0.4.2 引入新语言 adapter。

## 成功标准

- 三个 capsule 均可 `manifest`、`inspect`、`check`、`test`、`serve --mcp --dry-run` 和 `pack`。
- 文档清楚解释官方胶囊的信任边界。
- README 仍然围绕当前实现状态，不把愿景文档当作当前承诺。
- 0.5 的 Language Agnosticism 留在后续架构讨论，不进入 0.4.2 runtime scope。
