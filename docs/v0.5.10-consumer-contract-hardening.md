# v0.5.10 Consumer Contract Hardening

## 一句话判断

v0.5.10 应该是 Desktop 前的关键 Core 合同校准：不做 UI，不扩展多客户端 apply，不引入 daemon，而是把 README、Router、mount、run history 和 consumer JSON 的公开边界稳定下来。

## 背景

v0.5.8 已经提供本地 SkillRun Router：

```bash
skillrun router serve --mcp
skillrun router serve --mcp --dry-run
```

v0.5.9 已经提供 Claude Desktop Safe Mount Apply / Rollback：

```bash
skillrun consumer mount apply --client claude-desktop --json
skillrun consumer mount rollback --client claude-desktop --backup <path> --json
```

这意味着 v0.6 Desktop 不再缺“能否挂载”的底座，真正缺的是稳定、诚实、可消费的 Core 合同。

## 核心原则

### 1. Desktop consumes contracts, not internals

Desktop 只能消费 CLI JSON 合同和显式命令，不应读取 `.skillrun/`、registry 文件、run record 文件或 MCP 文本输出作为稳定接口。

### 2. Core owns side-effect semantics

Core 负责本地配置写入、backup、rollback、Consumer Mode validation、Router exposure 和 run evidence 的语义。Desktop 负责 consent、展示、过滤和交互。

### 3. No new trust claims

v0.5.10 不把 mount apply、registry、switchboard 或 `.skr` 描述成 sandbox、secure install、trust store 或 marketplace。

## 目标

v0.5.10 的目标是：

- 清理 README 和版本文档中已过期的计划态表述。
- 让 `consumer mount plan` 展示的 backup path pattern 与 `apply` 实际 backup 命名一致。
- 保持 Claude Desktop 为唯一 apply / rollback 支持对象。
- 明确 v0.6 Desktop alpha 应优先消费现有 CLI JSON surface。
- 识别但不急于实现 Desktop 需要的后续 Core surface，例如 `consumer runs inspect --json` 和 `.skr import`。

## 非目标

v0.5.10 不做：

- Desktop / Tauri。
- `skillrun ui`。
- daemon API。
- Router hot reload。
- Router process management。
- Cursor apply。
- 多客户端 mount adapter。
- `.skr import`。
- marketplace。
- signed package trust。
- dependency installation。
- OS sandbox。
- 大规模模块重构。

## 成功标准

- README 和中文 README 不再把已实现 Router 描述成未来能力。
- v0.5.8 / v0.5.9 文档从计划态更新为实现态，并保留后续演进边界。
- v0.6 愿景文档反映 v0.5.8 / v0.5.9 已完成事实。
- `consumer mount plan --json` 的 backup path pattern 与 `consumer mount apply --json` 的 backup 文件命名格式一致。
- mount plan / apply / rollback 测试继续覆盖 plan-only、noop、rollback conflict 和错误备份边界。
- `cargo test` 和 `cargo clippy --all-targets -- -D warnings` 通过。

## 后续建议

- v0.5.11：增加 `consumer runs inspect <run-id> --json`，支撑 Desktop Envelope Explorer。
- v0.5.12：增加 `.skr import` 或等价本地 capsule import flow，支撑 Desktop 官方包导入。
- v0.6.0：在独立 `skillrun-desktop` 项目中做 Tauri alpha，消费 Core JSON 合同，不复制 Core 语义。
