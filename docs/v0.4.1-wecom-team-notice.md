# SkillRun v0.4.1: WeCom Team Notice Example

**文档状态**：Draft  
**版本**：v0.4.1 example design  
**最后更新**：2026-05-13  
**来源**：用户希望跑通“本地发企业微信通知”的正式可用示例。

---

## 一句话判断

`wecom_team_notice` 应该是一个正式可运行的 Skill Capsule 示例，而不是企业微信 API wrapper、OpenAPI-to-MCP demo 或企业微信 adapter。

它验证的是：

> 一个团队通知 SOP 可以被打包成可 dry-run、可审批、可审计、可通过 MCP 调用的本地 Agent skill。

## 示例定位

`wecom_team_notice` 面向这样的用户：

- 想让本地 AI assistant 帮自己给企业微信群发通知。
- 想先预览通知内容，再决定是否真实发送。
- 想避免 Agent 直接调用一个裸的 `send_message(text)` 工具。
- 想把通知 SOP、输入 schema、审批边界、env 权限和运行证据一起分发。

这个示例要让用户 5 分钟内理解两种路径：

```text
人本地执行：
  skillrun run --cwd examples/wecom_team_notice --input examples/dry_run.input.json

Agent 调用：
  MCP client starts:
    skillrun serve --mcp --cwd examples/wecom_team_notice
  Agent sees the Manifest-derived tool and calls it through MCP tools/call.
```

Agent 不应该自己猜测或手写 `skillrun run` 命令。Agent 的稳定调用入口是 MCP tool。`SKILL.md` 作为 MCP resource 和 tool description 的来源，负责让 Agent 理解 SOP、禁止场景和恢复策略。

## 为什么不是 FastMCP 示例

FastMCP 风格的最小工具可能是：

```python
def send_wecom_message(text: str) -> str:
    ...
```

SkillRun 示例必须展示更高一层的产品原子：

```text
wecom_team_notice =
  团队通知 SOP
  + typed input/output
  + dry-run preview
  + approval boundary
  + WECOM_WEBHOOK_URL declared env
  + structured DependencyError / PolicyViolation
  + markdown notice artifact
  + run record
  + Manifest-derived MCP tool
```

核心不是“能发一条企业微信消息”，而是“能把团队通知发布能力作为 Skill Capsule 分发和消费”。

## Capsule 形态

```text
examples/wecom_team_notice/
  SKILL.md
  action.py
  skillrun.config.json
  examples/
    dry_run.input.json
    send.input.json
    urgent_requires_approval.input.json
    invalid_secret.input.json
  .skillrun/
    manifest.generated.yaml
```

首版使用 Python `action.py`，不使用 bash 作为 action adapter。

原因：

- Python + Pydantic 能稳定表达 typed input/output。
- `preflight` 能清晰表达审批和禁发边界。
- structured error envelope 和 artifact helper 更贴近 SkillRun 核心价值。
- Windows 用户不应被 bash 运行环境卡住。
- bash 示例容易把 SkillRun 降级成 shell wrapper。

## 输入合同

建议输入：

```text
title: string
summary: string
audience: "team" | "project" | "incident" | "all_hands"
urgency: "normal" | "high" | "critical"
dry_run: boolean
approval_id: optional string
mentioned_mobile_list: optional string[]
```

约束：

- `dry_run=true` 不需要企业微信 webhook。
- `dry_run=false` 必须存在 `WECOM_WEBHOOK_URL`。
- `urgency=high` 或 `critical` 时必须有 `approval_id`。
- `audience=all_hands` 时必须有 `approval_id`。
- 通知内容必须有长度上限。
- 禁止发送明显 secrets，例如 `sk-`、`AKIA`、`BEGIN PRIVATE KEY`、`password=`、`token=`。

## 输出合同

建议输出：

```text
decision: "preview" | "sent" | "blocked"
message_preview: string
wecom_response: optional object
audit_note: string
```

artifact：

- `notice.md`：记录 title、audience、urgency、dry_run、approval_id、message preview、发送结果或阻断原因。

## 错误语义

- `PolicyViolation`：缺审批、疑似 secret、超出内容边界、禁止 audience。
- `DependencyError`：`dry_run=false` 但缺少 `WECOM_WEBHOOK_URL`，或企业微信 webhook 调用不可达。
- `ValidationError`：schema 不匹配，例如 urgency 不在枚举里。
- `RuntimeError`：未分类失败，保留低层日志，但不把 stack trace 暴露给 Agent。

## 权限与配置

`skillrun.config.json` 应声明：

```json
{
  "runtime": {
    "timeout": "30s",
    "env": ["WECOM_WEBHOOK_URL"]
  },
  "permissions": {
    "network": {
      "outbound": ["qyapi.weixin.qq.com"]
    },
    "env": {
      "read": ["WECOM_WEBHOOK_URL"]
    }
  }
}
```

v0.4.1 不要求 SkillRun Core 强制网络 sandbox。文档必须继续诚实说明：权限声明是运行契约和 inspect/check 信息，不是 OS/network sandbox。

## 本地使用路径

### 预览通知

```bash
skillrun manifest --cwd examples/wecom_team_notice
skillrun inspect --cwd examples/wecom_team_notice
skillrun check --cwd examples/wecom_team_notice
skillrun run --cwd examples/wecom_team_notice --input examples/dry_run.input.json
```

### 真实发送

PowerShell：

```powershell
$env:WECOM_WEBHOOK_URL="https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=..."
skillrun run --cwd examples/wecom_team_notice --input examples/send.input.json
```

真实发送必须满足：

- `dry_run=false`
- `WECOM_WEBHOOK_URL` 已声明并注入
- 高优先级或全员通知有 `approval_id`

## Agent / MCP 使用路径

MCP client 配置示例：

```json
{
  "mcpServers": {
    "skillrun-wecom-team-notice": {
      "command": "skillrun",
      "args": [
        "serve",
        "--mcp",
        "--cwd",
        "D:/path/to/skillrun/examples/wecom_team_notice"
      ],
      "env": {
        "WECOM_WEBHOOK_URL": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=..."
      }
    }
  }
}
```

给 Agent 的推荐指令：

```text
请使用 wecom_team_notice MCP tool 发送团队通知。
先 dry_run 预览；如果返回 preview，我确认后再把 dry_run 改成 false。
不要自己调用 skillrun run。通过 MCP tool 调用，并遵守 SKILL.md 的审批和禁发规则。
```

## v0.4.1 成功标准

- `examples/wecom_team_notice` 是完整可运行 capsule。
- `dry_run.input.json` 不需要真实 webhook，也能通过 `skillrun test`。
- `send.input.json` 在配置 `WECOM_WEBHOOK_URL` 后可以真实发送。
- `urgent_requires_approval.input.json` 返回 `PolicyViolation`。
- 缺少 `WECOM_WEBHOOK_URL` 时，真实发送返回结构化 `DependencyError`。
- `skillrun serve --mcp --cwd examples/wecom_team_notice --dry-run` 暴露稳定 tool schema。
- README 或 docs 能说明本地 CLI 和 Agent/MCP 两条路径。
