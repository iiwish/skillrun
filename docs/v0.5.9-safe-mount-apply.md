# v0.5.9 Safe Mount Apply

## 一句话判断

v0.5.9 应该把 `consumer mount plan` 推进到一个极窄、可逆、可审计的 `apply / rollback` MVP：Core 负责 mount semantics，Desktop 只负责 consent UX。

## 背景

v0.5.6 已经实现 plan-first 的 MCP client 配置预览：

```bash
skillrun consumer mount plan --client <id> [--config <path>] --json
```

v0.5.8 已经实现真实 Router runtime：

```bash
skillrun router serve --mcp
```

这意味着 `mount plan` 现在指向的运行入口已经存在。v0.5.9 的问题不再是“挂谁”，而是“如何把 plan 变成一次可拒绝、可验证、可回滚的本地配置写入”。

## 核心原则

### 1. Core owns Mount Semantics

`skillrunv2` 应负责：

- MCP client config 的结构化读取。
- SkillRun Router entry 的 patch 规则。
- backup metadata。
- idempotency。
- rollback。
- conflict detection。
- JSON contract。
- contract tests。

### 2. Desktop owns Mount Consent

`skillrun-desktop` 应负责：

- 展示 plan diff。
- 明确请求用户确认。
- 调用 `skillrun consumer mount apply ... --json`。
- 展示 apply 结果。
- 提供 rollback 按钮。

Desktop 不应直接读写 MCP client config，也不应复制 Core 的 patch 逻辑。

## 目标

v0.5.9 的最小目标：

```bash
skillrun consumer mount apply --client claude-desktop [--config <path>] --json
skillrun consumer mount rollback --client claude-desktop --backup <path> [--config <path>] --json
```

实现约束：

- 只支持 `claude-desktop` 的 apply / rollback。
- `cursor` 继续保留 plan-only。
- apply 必须复用 mount plan 计算出的 patch。
- apply 必须先写 backup，再写 config。
- apply 必须支持 missing config：创建父目录和最小 JSON config。
- apply 必须 idempotent：已经是目标 entry 时不重复写无意义 backup。
- rollback 只能回滚 SkillRun 自己创建的 backup。
- rollback 必须检测当前 config 是否仍可安全恢复；如果用户在 apply 后修改了同一 `skillrun` entry，应 fail closed。
- 输出不得泄露非 SkillRun server 的 raw config、env、token 或 command args。
- backup 文件会保存原始 config 的完整 JSON，用于精确 rollback；它是本地敏感文件，不应被 Desktop 当作可公开展示内容。

## 非目标

v0.5.9 不做：

- Desktop / Tauri。
- `skillrun ui`。
- daemon API。
- Router hot reload。
- 启动或管理 Router 进程。
- 支持 Cursor apply。
- 多客户端 adapter 扩展。
- `.skr import`。
- marketplace。
- signed package trust。
- dependency installation。
- OS sandbox。
- “安全安装”叙事。

## 失败边界

Apply 必须 fail closed：

- unsupported client。
- `cursor` apply / rollback。
- config root 不是 JSON object。
- `mcpServers` 不是 JSON object。
- config 无法解析。
- backup 写入失败。
- config 写入失败。

Rollback 必须 fail closed：

- backup 不存在。
- backup 不是 SkillRun 创建的 backup。
- backup 目标 config 与当前 `--config` 不匹配。
- 当前 config 无法解析。
- 当前 `skillrun` entry 已经不是 SkillRun Router entry。
- backup payload 无法还原成安全 JSON object。

## 成功标准

- `consumer mount apply --client claude-desktop --config <path> --json` 可以创建 missing config。
- apply 后 config 中存在 `mcpServers.skillrun = {"command":"skillrun","args":["router","serve","--mcp"]}`。
- apply 会创建 SkillRun backup 文件。
- 对已经挂载的 config 再次 apply 返回 noop，不创建新 backup。
- rollback 可以恢复 apply 前的 config。
- rollback 在 `skillrun` entry 被用户修改后拒绝覆盖。
- unsupported / cursor apply 返回结构化 warning，不写文件。
- README、release notes 和 docs index 明确 v0.5.9 是 Safe Mount Apply，不是 Desktop 或 daemon。

## 实现状态

v0.5.9 已完成并合入 `main`：

- CLI 已提供 `consumer mount apply` 和 `consumer mount rollback`。
- `apply` 复用 `mount plan` 的 patch 语义，只对 `claude-desktop` 执行写入。
- `rollback` 只接受 SkillRun 自己创建的 backup，并在当前 `skillrun` entry 被用户修改后 fail closed。
- 集成测试覆盖 missing config、已有 config、noop、rollback conflict、missing/invalid backup、unparseable current config 和 Cursor plan-only apply。
- README、release notes 和 docs index 已明确 v0.5.9 是 Safe Mount Apply，不是 Desktop、daemon 或多客户端配置管理。

## 后续演进

- 在扩展更多 MCP client apply 前，先保持 Claude Desktop 路径稳定。
- Desktop 只能展示 backup 路径和回滚入口，不应展示 backup 文件的完整 payload。
- 后续版本应继续收紧 backup path 展示、config identity 和本地写入失败边界。
