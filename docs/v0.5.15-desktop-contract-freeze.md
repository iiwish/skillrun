# v0.5.15 Desktop Contract Freeze

**状态**：In_Progress  
**日期**：2026-05-18  
**分支**：`codex/v0.5.15-desktop-contract-freeze`

## 一句话判断

v0.5.15 应冻结 Desktop alpha 可以绑定的 Core 合同：不做 Desktop、不做 daemon、不扩 Router，只让 tray-first Desktop 不必依赖 stderr 字符串或隐含版本判断。

## 背景

v0.5.14 增加了 `skillrun host status --json`，Desktop 已经可以在启动时确认 Core 可用性、版本、平台、路径和 capability 列表。

但 Desktop alpha 仍有两个契约风险：

- `host.status.v1` 没有显式声明 Desktop contract set，UI 只能从二进制版本间接推断兼容性。
- `skillrun import --json` 在运行期失败时仍只输出 stderr，Desktop 必须解析人类错误文本才能判断 duplicate id、非法包路径等场景。

## 范围

### 1. Desktop Contract Set

`skillrun host status --json` 增加：

```json
{
  "desktop_contract": {
    "name": "desktop.alpha",
    "version": 1,
    "status": "frozen"
  }
}
```

这不是新的 daemon API，也不是 UI API；它只是告诉 Desktop 当前 Core 支持哪一组稳定 CLI JSON surfaces。

### 2. Import JSON Error Contract

当用户调用：

```text
skillrun import <package.skr> --json
```

如果运行期失败，Core 仍返回非零 exit code，但 stdout 返回结构化 JSON：

```json
{
  "command": "import",
  "schema_version": "import.v1",
  "ok": false,
  "error": {
    "code": "registry-id-exists",
    "message": "registry id already exists: refund"
  },
  "warnings": []
}
```

首版稳定 error code：

- `registry-id-exists`
- `import-target-exists`
- `package-not-found`
- `package-not-file`
- `package-path-escape`
- `unsupported-package-entry`
- `invalid-manifest`
- `import-failed`

解析参数失败仍是 CLI usage error，不属于 import runtime JSON contract。

## 非目标

v0.5.15 不做：

- Desktop / Tauri / tray implementation。
- daemon API。
- Router process management。
- Router hot reload。
- 全局 JSON error framework。
- artifact/log/input content read。
- signed trust。
- dependency installer。
- marketplace。

## Desktop 使用规则

Desktop alpha 应：

- 先读 `host.status.v1.desktop_contract`。
- 只绑定 `desktop.alpha` / version `1` 中列出的 capability surfaces。
- 对 `import --json` 同时检查 exit code 和 stdout JSON。
- 对 import failure 使用 `error.code`，不解析 stderr。
- 继续把 parse/usage error 视为 Desktop caller bug 或用户命令构造错误。

## 验收标准

- `host status --json` 返回 `desktop_contract.name="desktop.alpha"`、`version=1`、`status="frozen"`。
- host status fixture 覆盖 `desktop_contract`。
- `import --json` duplicate registry id 返回 `ok=false` JSON 和 `error.code="registry-id-exists"`。
- `import --json` package path escape 返回 `ok=false` JSON 和 `error.code="package-path-escape"`。
- README / docs index / release notes 说明 v0.5.15 是 Desktop Contract Freeze，不是 Desktop release。
