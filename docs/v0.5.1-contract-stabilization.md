# SkillRun v0.5.1: Contract Stabilization

**文档状态**：Planning Contract
**版本**：v0.5.1 planning baseline
**最后更新**：2026-05-15

---

## 一句话判断

v0.5.1 不是新功能版本，而是把 v0.5.0 的协议、信任模型、官网叙事和 envelope 字段收敛成一套一致的公开合同。

v0.5.0 已经证明 Adapter Protocol 和 Level 0 command adapter；v0.5.1 要证明的是：

> **SkillRun 的边界叙事和机器合同一样稳定。**

## 为什么需要 v0.5.1

v0.5.0 之后，项目已经从单语言 MVP 进入协议阶段。此时继续叠加 UI、Router、registry 或官方领域包会过早放大不一致：

- SSOT、README、官网和 Adapter Protocol 对 release 状态的描述需要统一。
- success envelope 的规范字段必须稳定为 `output`，不能继续在文档中混用 `result`。
- “无护栏，不执行”可以作为强叙事，但必须定义 guardrail 的真实含义。
- trust model 必须继续明确：SkillRun 减少隐式执行、隐式成功和隐式信任，但不是 OS sandbox。
- v0.6 Consumer Era 需要 `--json`、registry、router 和 run query 等 headless 能力；这些能力不能建立在不稳定合同上。

## 规范结论

### Envelope 字段

成功 envelope 的规范字段是：

```json
{
  "ok": true,
  "output": {},
  "artifacts": [],
  "display": {
    "markdown": "Done."
  }
}
```

`output` 必须符合 Manifest 中的 output schema。

`result` 只能作为历史文档或作者侧兼容输入出现，不应作为 Core-to-adapter 标准 envelope 字段写入新文档、新示例或官网。

### Guardrail 定义

SkillRun 的 guardrail 不是抽象营销词，也不是 sandbox。

在 SkillRun 中，guardrail 指：

```text
Manifest contract
+ input/output schema
+ preflight checks
+ structured output/error envelope
+ artifact containment
+ run evidence
+ Consumer Mode static checks
```

可以说：

> **No guardrail, no execution.**

但必须解释为：

> 没有 Manifest、schema、preflight 和 structured envelope 形成的代码级边界，Agent 不应直接执行 action。

不能说：

- SkillRun 能安全运行任意第三方代码。
- SkillRun 提供 OS sandbox。
- `.skr` 是 secure install format。
- dependency readiness 等于可复现运行环境。
- command adapter 是安全 shell runner。

### Trust Model

v0.5.1 继续承诺小而硬的边界：

- 不隐式执行 instruction-only skill。
- 不从 stdout 推断成功。
- Consumer Mode 不为 metadata 动态 import 未信任源码。
- Manifest 缺失、过期或 source hash 不匹配时 fail closed。
- declared env 和 artifact path 是运行合同的一部分。

v0.5.1 不新增安全承诺：

- 不做 OS sandbox。
- 不做网络强隔离。
- 不做 dependency vendoring。
- 不做 signed package trust。
- 不做 registry 审核。

## 文档与官网统一口径

推荐中文主叙事：

> **无护栏，不执行。**
>
> SkillRun 把 SOP、schema、preflight、structured envelope 和运行证据绑定到 Action 上。Agent 调用的不再是裸函数，而是可检查、可拒绝、可追溯的 Skill Capsule。

推荐英文主叙事：

> **No guardrail, no execution.**
>
> SkillRun turns SOP-backed actions into manifest-bound skills with preflight checks, structured envelopes, artifacts, and auditable run evidence.

避免使用：

- “彻底终结幻觉灾难”
- “防弹装甲”
- “绝对安全”
- “目录级隔离”等可能暗示 OS sandbox 的表达
- “任何语言都能无缝接入”这类超过 Level 0 command adapter 边界的表达

## v0.5.1 验收标准

- README、README.zh-CN、SSOT、trust model、Adapter Protocol 和官网对 guardrail 的定义一致。
- 新文档中的 success envelope 使用 `output`，不使用 `result`。
- 官网保留 “No guardrail, no execution / 无护栏，不执行” 的尖锐表达，但明确非 sandbox 边界。
- v0.6 Consumer Era 被描述为计划能力，不被写成已实现能力。
- 没有新增 runtime 行为或 CLI 合同。

## 后续版本衔接

v0.5.1 之后的合理顺序：

```text
v0.5.2  Consumer JSON Surface
v0.5.3  Capsule Registry + Switchboard
v0.5.4  SkillRun Router + Mount Profiles
v0.5.5  Official Pack Foundations
v0.6.0  Consumer Era local console
```

v0.5.1 的价值在于让这些后续能力有一个稳定、诚实、可复述的合同地基。
