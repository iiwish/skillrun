# SkillRun v0.6 Skill Capsule Contract

**文档状态**：Contract freeze candidate
**版本**：v0.6 planning baseline
**最后更新**：2026-05-20

---

## 一句话判断

v0.6 的核心不是把 SkillRun 做成 MCP 工具管理器，而是冻结 **Skill Capsule** 作为 Agent Skill 基础单元的合同。

> SkillRun 是面向 Agent Skill 的开放运行时：让技能可检查、可运行、可约束、可审计、可分发。

MCP 是连接层和导出层。SkillRun 的核心模型必须先于 MCP 存在，也必须能被 Desktop、CLI、Router、CI 和未来 registry 共同消费。

## Contract 分层

SkillRun v0.6 把一个可消费技能分成六层：

```text
Source Capsule
  SKILL.md + action code + examples + skillrun.config.json

Manifest Contract
  generated runtime contract, source hashes, schemas, permissions, adapter

Package
  .skr source + Manifest archive

Local Inventory
  registry + switchboard enabled state

Exposure
  Router-derived MCP tools/resources from enabled capsules only

Run Evidence
  structured envelope + artifacts + logs + run record
```

任何新能力都必须说明自己属于哪一层。不能让 Desktop、MCP、registry 或 installer 反向定义 Manifest、Adapter Protocol 或 run evidence 的语义。

## 最小 Capsule 结构

一个 v0.6 Skill Capsule 至少包含：

```text
capsule/
  SKILL.md
  action.py | action.mjs | command entrypoint
  examples/
    default.input.json
  skillrun.config.json       # 推荐显式声明；可由 convention 补齐
  .skillrun/
    manifest.generated.yaml
```

最小合同：

- `SKILL.md` 是 SOP 和使用边界，不是装饰性 README。
- action entrypoint 必须显式存在；SkillRun 不从 Markdown、references、assets 或 examples 推断可执行动作。
- input/output schema 必须能被 Core 静态读取或在 Author Mode 生成到 Manifest。
- examples 至少提供一个默认输入，用于 `check`、`test`、文档和 conformance。
- Manifest 必须绑定 source hashes；Consumer Mode 发现 Manifest 缺失、过期或 hash 不匹配时 fail closed。

## Manifest Contract

v0.6 不要求立刻提升 Manifest IR 版本。当前 `manifest_version = 0.1.0` 可以继续存在，但下列字段进入 contract freeze 范围：

- `manifest_version`
- `generated_by`
- `sources.skill/action/config.path`
- `sources.*.sha256`
- `skill.name`
- `skill.sop_summary`
- `tool.name`
- `tool.description`
- `schemas.input`
- `schemas.output`
- `runtime.adapter`
- `runtime.entrypoint`
- `runtime.command`
- `runtime.protocol_version`
- `runtime.requirements`
- `permissions`
- `ipc.protocol_version`
- `examples`
- `artifacts.allowed_kinds`
- `errors.envelope`

规则：

- Manifest 是 runtime authority；Consumer Mode 不 import source 来重新发现 schema。
- `tool.description` 是 Agent 软约束；真正的边界必须落在 schema、permissions、preflight 和 envelope 中。
- `runtime.protocol_version = adapter.v1` 标识 adapter-facing 合同；`ipc.protocol_version = 0.1.0` 标识 Core 文件 IPC 合同。
- Manifest 字段可以扩展，但不能让现有 Desktop alpha JSON consumer 依赖未冻结的内部目录结构。

## Adapter Protocol v1

v0.6 继续承认三类 adapter：

| Level | 名称 | 维护边界 |
| --- | --- | --- |
| 0 | command adapter | 任意进程只要遵守 IPC/envelope，即可运行 |
| 1 | community adapter | 社区维护，通过 conformance fixtures |
| 2 | blessed adapter | 官方维护，有模板、文档、CI 和 release tests |

Adapter Protocol v1 的冻结要求：

- stdout/stderr 永远是日志，不是结构化结果。
- 成功必须写入 `SKILLRUN_OUTPUT_JSON`，并产生 `ok: true` envelope。
- 失败必须产生 structured error envelope，或由 Core 映射成 `ProtocolViolation` / `DependencyError` / `PermissionDenied` 等稳定错误。
- artifact 必须写入 `SKILLRUN_ARTIFACT_DIR` 内，并由 Core 校验路径不越界、文件存在。
- metadata phase 只属于 Author Mode，不注入业务 secrets，不执行业务 action。
- Consumer Mode 只读 Manifest，不运行 metadata phase。

## Consumer JSON Surface

v0.6 的 Desktop、automation 和 worker 不应读取 `.skillrun/` 内部文件作为稳定 API。稳定消费入口是 CLI JSON surface：

- `skillrun inspect --json`
- `skillrun check --json`
- `skillrun doctor --json`
- `skillrun import <package.skr> --json`
- `skillrun registry list --json`
- `skillrun switchboard list --json`
- `skillrun consumer inventory --json`
- `skillrun consumer exposure --json`
- `skillrun consumer mount plan --client <id> --json`
- `skillrun consumer mount apply --client <id> --json`
- `skillrun consumer mount rollback --client <id> --backup <path> --json`
- `skillrun consumer runs list --json`
- `skillrun consumer runs inspect <run-id> --json`
- `skillrun host status --json`

新增 Desktop 或 website 功能时，优先扩展这些 JSON surface，而不是复制 Core 逻辑。

## Package 与 Import

`.skr` 是 source + Manifest archive。

它不是 secure install format、OS sandbox、dependency bundle、signed trust artifact 或 marketplace listing。

v0.6 的 import 合同：

- import 必须校验 package 结构和 Manifest。
- imported capsule 进入本地 registry。
- enable/disable 状态属于 switchboard，不属于 `.skr`。
- Router 只暴露 enabled capsules。
- MCP client 应挂载 SkillRun Router，而不是直接挂载 `.skr` 或 capsule 文件夹。

## Run Evidence

v0.6 的 run evidence 至少由这些部分组成：

- run id
- mode：`test` / `run` / `mcp`
- started/finished/duration
- capsule path 或 imported capsule id
- Manifest hash
- skill/action/config source hash
- declared permissions snapshot
- stdout/stderr log availability
- output/error envelope
- artifact metadata

默认不承诺返回完整 input、artifact 内容、stdout/stderr 全文、全局 run index、retention policy 或 PII redaction policy。这些可以作为后续能力加入，但不能在没有明确 contract 前让 Desktop 或 website 依赖内部 evidence 文件布局。

## Fail-Closed Rules

v0.6 必须继续 fail closed：

- 缺失 Manifest：不执行。
- Manifest stale：不执行。
- source hash mismatch：不执行。
- schema invalid：不执行。
- input invalid：不执行 action。
- undeclared env access：不注入。
- missing runtime executable：返回 `DependencyError`。
- malformed output envelope：返回 `ProtocolViolation`。
- artifact path traversal：返回 `PermissionDenied`。
- disabled capsule：不暴露为 MCP tool。

这些规则比“工具描述写得好”更重要。Agent 只能把 description 当提示，Core 必须把合同当边界。

## v0.6 验收清单

v0.6 Contract Freeze 的完成条件：

- README 和中文 README 明确 SkillRun 是 Skill Capsule runtime，不是 MCP manager。
- Manifest contract 字段有 fixture 或 snapshot 覆盖。
- Adapter conformance suite 覆盖 Python、JS alpha 和 command adapter 的核心行为。
- Consumer JSON surface 有 fixture 覆盖，并在 Desktop 中只通过这些 surface 消费 Core。
- Release 后有 installer、archive、latest 和 asset guard。
- `.skr` import、registry、switchboard、Router exposure 和 run evidence 在文档中形成一条完整链路。
- Trust/provenance 文档明确 checksum、signature、publisher metadata 和 sandbox 的边界。

## 非目标

v0.6 不承诺：

- marketplace
- hosted registry
- OS sandbox
- signed package enforcement
- dependency vendoring
- automatic Python/Node package installation
- daemon API
- Router hot reload
- global run retention policy
- Desktop 内部重写 Core 语义

这些都可以进入 v0.7+，但不能阻塞 v0.6 的核心合同冻结。

## 后续工作图

建议按以下顺序推进：

1. **Contract fixtures**：把 Manifest、Consumer JSON surface、run evidence 的关键字段纳入 fixtures。
2. **Adapter conformance suite**：让 Python / JS / command adapter 都证明自己遵守 `adapter.v1`。
3. **Provenance model**：设计 checksum、publisher、signature、trust state，但不夸大成 sandbox。
4. **Desktop alpha**：只消费冻结 JSON surface，完成 Import -> Switchboard -> Exposure -> Mount -> Runs。
5. **Website quickstart**：以 5 分钟跑通 Skill Capsule 为核心，不做空泛营销页。
